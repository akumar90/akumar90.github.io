SOCIAL GYM - AN IOT PROJECT
Transform gyms into a social hub, by making it a ‘smarter’ place
Product Management Internet Of Things

What exactly was my role in the project...
I was acting as a Product Manager to define the distinct use cases, and personas for which such a solution has to be built.
In addition to this, I was also responsible for the architectural design of the solution, as well as writing some of the code to make this solution a reality. We also perfomed user testing to improve our design, and also take into consideration edge cases where the equipments were not always present where they should be, and design for such use cases.
What were our accomplishments...

  • We associated beacons with three pieces of exercise equipment, associating each piece of equipment with a unique ID (with both human-readable and machine-readable versions).

  • Using the Physical Web application, they can access a web page and “check-in” or “check-out” from the equipment that corresponds to that beacon. Both the duration and type of workout are linked to the user’s profile.

  • Finally, the system uses Twilio to send the user a text message from one of the pieces of equipment with various recommendations on when and how to use it.

What principles were used in this project...
  • Web-based Solution

    We wanted to avoid the need for users to download a dedicated application for our service. Instead, it should be accessible through a standard mobile web browser.

  • Intelligence as a Service

    Rather than baking intelligence into the product itself, we wanted to create a solution that could adapt to any gym and give its equipment even rudimentary intelligence.

How were the interactions being handled...
  • Equipment Pages

    Each piece of gym equipment is associated with a web page, which users can use to check in and check out of equipment. This is the primary interaction, and over time, helps the gym set up an association between a user and a piece of equipment (including time using the equipment in question.) Checking in and checking out changes an equipment model (inUse: true/false) which could eventually be stored in a database

  • Text Messaging

    As the gym starts to understand a person’s workout patterns, it can make intelligent recommendations for future workouts. We used the Twilio API to manage text messages, and ASBase to establish a more holistic picture of user activity. We know that physical health involves a variety of factors, like diet and schedule - so, by mashing up with other APIs and data sources, we can establish triggers and more holistic recommendations.

Some behind the scene details...
  • Equipment Schema

    We created a simple JSON schema for understanding equipment, which included an ID, a human-readable name, the equipment type, the workout that the equipment affords (i.e., strength, cardio, or flexibility) and whether or not the equipment is in use. The server uses this schema to nudge people towards more holistic, effective workouts.

  • Other Technologies

    The gym understands that its users might use other technologies; it publishes user activities through ASBase, and can ostensibly read from ASBase. Kind of. It does have a subscription to ASBase, and ASBase is sending it information.

What technologies were used in this project...
  • Node.js

    Featuring a robust ecosystem of tools, excellent flexibility for file structure, and native JavaScript (and, by extension, JSON) syntax, Node felt like an obvious choice for setting up our server.

  • Express

    Express is a standard tool for making the API more readable, and establishing standard server functionalities with less written code. This isn’t overly difficult to implement, but did trip us up a few times.

  • Twilio API

    Allowed us to send SMS with almost no technical investment on our end. This did, however, involve a minor monetary investment instead.

  • Activity Streams Web Standard (and ASBase)

    Offered us the opportunity to understand and interact with other services.

  • Heroku

    Used for deployment, especially testing the activity streams across IP addresses. By deploying on Heroku, we could avoid issues on the Cal network.

  • Eddystone Beacons

    Used as broadcasters associated with the equipments acting as endpoints.

SUMMARY
My Role...
  • Product Management
  • Requirements gathering as well as prioritization
  • Idea feasibility testing and market analysis
My accomplishments...

  • Successful implementation of our Social gym idea using various sensors and beacons, along with extensive user testing to improve our design.

  • Creation of an end-to-end flow which can be used to check-in and check-out equipments, and also receive text messages regarding instructions of how to use a certain equipment.
